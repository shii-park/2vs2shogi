## 開発環境の利用方法 (Usage)

本プロジェクトでは `make` コマンドを使用してDocker環境の操作を簡略化しています。

### 1\. 初回セットアップ

リポジトリをクローンした後、最初に一度だけ実行してください。
`.env` ファイルの作成、サブモジュールの初期化、Dockerイメージのビルドを一括で行います。

```bash
make setup
```

### 2\. アプリケーションの起動

開発を始める際は、以下のコマンドで全システム（Frontend, Backend, DB）を起動します。

```bash
make up
```

起動後、ブラウザで [http://localhost:3000](https://www.google.com/search?q=http://localhost:3000) にアクセスして確認してください。

### 3\. アプリケーションの停止

作業を終了する際は、以下のコマンドでコンテナを停止します。

```bash
make down
```

-----

### フロントエンド開発用コマンド

UI調整など、フロントエンド単体で作業を進めたい場合に使用します。

| コマンド | 説明 | 備考 |
| :--- | :--- | :--- |
| `make front-up` | フロントエンドのみ起動 | バックエンドとの通信が必要な機能は動きません |
| `make front-log` | ログの確認 | `Ctrl+C` で終了します |
| `make front-build` | 再ビルドして起動 | ライブラリ追加時や設定変更時に使用します |

-----

### 環境の完全リセット

Docker環境の調子が悪い時や、データを初期化したい場合に使用します。
**※データベースの内容（ボリューム）も全て削除されるため、注意して実行してください。**

```bash
make clear
```
